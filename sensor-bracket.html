Overview

This project simulates the full lifecycle of integrating a new sensor onto an aircraft — from mechanical design and structural analysis through software-driven data validation and center-of-gravity modeling. It bridges mechanical engineering fundamentals with Python-based data analysis in a workflow typical of aerospace operations roles.

The work was structured around three parallel tracks: physical design and simulation in OnShape, failure mode documentation using a formal FMEA process, and Python-based flight data analysis to surface anomalies in the simulated sensor output.

Bracket Design & Structural Analysis

The aluminum bracket was modeled in OnShape with material properties and mounting constraints typical of light aircraft sensor integrations. Design considerations included:

  • Material selection — 6061-T6 aluminum for its strength-to-weight ratio
  • Mounting geometry to minimize vibration coupling between sensor and airframe
  • Center-of-gravity shift calculation using mass moments: ΔCG = (m × d) / Mtotal
  • Stress concentration factor analysis at fastener holes

Failure Mode & Effects Analysis (FMEA)

A systematic FMEA was performed to identify and rank potential failure modes across the bracket, sensor, and mounting interface. Severity, occurrence, and detection ratings were used to calculate Risk Priority Numbers (RPN).

Failure Mode Effect Severity Occurrence Detection RPN Mitigation
Bracket fatigue fracture Sensor loss / FOD 9 3 4 108 NDT inspection interval
Fastener loosening Sensor misalignment 6 4 3 72 Torque-seal + pre-flight check
Corrosion at interface Structural degradation 5 3 5 75 Anodize coating + inspection
Sensor connector failure Data loss 6 2 2 24 Redundant connector routing
Vibration-induced drift Noisy sensor output 4 5 3 60 Vibration isolator mounts

Flight Data Simulation & Anomaly Detection

Synthetic flight data was generated to simulate realistic sensor output across normal and anomalous operating conditions. Python (NumPy, Pandas) was used to process the time-series data and flag statistical outliers.

  • Generated 10,000+ data points simulating altitude, vibration, and sensor voltage readings
  • Applied z-score thresholding to flag readings beyond 3σ from the rolling mean
  • Visualized anomalies with Matplotlib — identified 3 distinct anomaly clusters in the dataset
  • Exported annotated results to Excel for engineering review documentation
